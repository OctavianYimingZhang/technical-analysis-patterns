# Contributing

Thank you for considering contributing to this project! Here are some guidelines to help you get started.

## How to Contribute

### Reporting Issues
- Use GitHub Issues to report bugs, inaccuracies, or suggest improvements
- Clearly describe the issue and provide examples where possible
- For pattern rule corrections, cite the source material

### Pull Requests
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Commit with a clear message (`git commit -m "feat: add ascending triangle pattern"`)
5. Push to your fork (`git push origin feature/your-feature`)
6. Open a Pull Request

### What We're Looking For
- **Pattern additions**: New chart patterns with identification rules, confirmation criteria, and risk management
- **Corrections**: Fixes to existing pattern rules based on source material
- **Translations**: Translations of the skill into other languages
- **Examples**: Real-world pattern examples (without copyrighted chart images)
- **Framework integrations**: Connecting patterns with other analysis methods (Elliott Wave, Fibonacci, etc.)

### Style Guidelines
- Keep pattern descriptions concise and actionable
- Always include: identification rules, breakout confirmation, target measurement, stop-loss, and failure conditions
- Use consistent formatting with the existing patterns
- Cite source material where applicable

## Code of Conduct

Be respectful and constructive in all interactions. This is an educational project -- we welcome traders of all experience levels.
